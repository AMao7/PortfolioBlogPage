My first ever pipeline
project.

Hello World

I've never really written a blog anymore and I really dont know how, but what I will share with you the world or whoever is interested, my Journey into DevOps. This begins with my first ever project in DevOps - and that is building a fully functional pipeline for a JS app by hitting one command terraform init. Our job was to create a load balanced and autoscaled 2 tier architecture (for the app and db). The Architecture also needed to be a "Highly Available" application, meaning that it has redundancies across all three availability zones. A lot right, especially on my first big project, but more than anything I was excited and felt confident. Finally, this is a link.

Day 1

The first step was to create a Jenkins slave and master nodes for our pipelines For this particular project we (the team) utilised Chef as our configuration management tool, (I know, not ansible huh), but I wil use ansible for the next project! The first part of the CI/CD pipeline was then to run tests and cookbook builds in Jenkins automatically after any changes to the code (through webhooks) chef exec rspec spec . The next stage of our CI/CD pipeline is to create an AMI image using packer. I never have used packer before this and almost had no clue how I would do this. However I then realised packer is written in JSON (JavaScript Object Notation) and then i realised I've used this before scripting in python. Puppet in short is a configuration managment tool that is used for configuring to create AMI images for Amazon Web Services to run an instance that will serve as our environments .

AWS Shenanigians

The cloudformation tool used for this project was Amazon Web Services (AWS), before I completed my AWS Cloud Practiotiner Certification (a story I hope to share in the future). I almost had no clue the whats on hows on this service and its capabilities. I then realised whilst on this project, the capabilties of hosting environments on the cloud. The ability to scale (from one to 1000 instance) and automate processes is second to none. But anyways, I had to learn how to create an instance with a Virtual Private Cloud first before even trying to automate it using puppet.

The first blunder

A week left to get this done and we just couldn't the NodeJs app and mongodb database to "seed" through terraform scripting. After actual days of colloboration we finally managed to figure it and yep it because they were on different ports :D. Despite this loss of time and our frustruation, I was positive that we would get this project done in time, after all - who doesnt love a challenge.

Monitoring using the ELK stack

s Immutable architectures are notoriously difficult to debug because we no longer have access to the instances and thus do not have access to the logs for those machines. Log consolidation allows us to have logs files broadcast to a central repository by the instances themselves which allows us to more easily view them. The ELK stack is a commonly used system for this purpose. Elk is made up of Elastic Search, Logstash and Kibana and also Beats.
